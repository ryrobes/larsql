# RVBBIT Documentation Site

This directory contains a static HTML documentation site for RVBBIT, built from actual codebase introspection rather than outdated markdown files.

## Overview

The documentation is built as **raw HTML** with shared CSS assets, styled to match the RVBBIT landing page (v3). This approach provides:

- ✅ Perfect style matching with landing page
- ✅ No build dependencies or tooling required
- ✅ Works immediately when deployed (just serve the files)
- ✅ Full control over styling and layout
- ✅ Fast, static pages with no JavaScript dependencies

## Design Principles

### Style Guide

The documentation follows the RVBBIT landing page aesthetic:

- **Color Palette:**
  - Background: `#05070c` (deep space black)
  - Panel: `#0b0f1a` (dark panel)
  - Cyan: `#1cf6ff` (primary accent)
  - Magenta: `#ff2fa8` (secondary accent)
  - Green: `#00ff88` (success/tips)
  - Orange: `#ffbd2e` (warnings)

- **Typography:**
  - Headings: Space Grotesk
  - Code: JetBrains Mono
  - Body: Space Grotesk with fallback to system fonts

- **Components:**
  - Code blocks with language badges
  - Info boxes (default, warning, tip variants)
  - Syntax-highlighted code snippets
  - Responsive tables
  - Sticky sidebar navigation

### Content Principles

1. **Accuracy First** - All examples and configurations verified against actual source code
2. **Practical Examples** - Every concept demonstrated with working code
3. **Progressive Disclosure** - Start simple, add complexity gradually
4. **Cross-Linking** - Related concepts linked throughout
5. **Real-World Use Cases** - Show when and why to use each feature

## Deployment

### Local Development

Simply open any HTML file in a browser:

```bash
# macOS
open doc_pages/index.html

# Linux
xdg-open doc_pages/index.html

# Or use Python's built-in server
cd doc_pages
python3 -m http.server 8000
# Visit http://localhost:8000
```

### Production Deployment

#### Option 1: Static Hosting (Recommended)

Deploy to any static hosting service:

**Netlify:**
```bash
# Drop the doc_pages folder in Netlify's web UI
# Or use Netlify CLI:
netlify deploy --dir=doc_pages --prod
```

**Vercel:**
```bash
# In project root:
vercel --prod
# Point to doc_pages directory
```

**GitHub Pages:**
```bash
# Copy doc_pages contents to gh-pages branch
git checkout --orphan gh-pages
cp -r doc_pages/* .
git add .
git commit -m "Deploy documentation"
git push origin gh-pages
# Enable GitHub Pages in repo settings
```

**AWS S3 + CloudFront:**
```bash
# Sync to S3 bucket
aws s3 sync doc_pages/ s3://your-bucket-name/ --delete
# Configure bucket for static website hosting
# Optionally add CloudFront distribution
```

#### Option 2: Integrate with RVBBIT Studio

Serve docs from RVBBIT's web server:

```python
# In studio/backend/app.py, add route:
@app.route('/docs')
@app.route('/docs/<path:page>')
def docs(page='index.html'):
    return send_from_directory('../doc_pages', page)
```

Then access at `http://localhost:5050/docs/`

#### Option 3: Nginx/Apache

Serve as static files:

**Nginx:**
```nginx
server {
    listen 80;
    server_name docs.rvbbit.io;
    root /path/to/rvbbit/doc_pages;
    index index.html;

    location / {
        try_files $uri $uri.html $uri/ =404;
    }
}
```

**Apache:**
```apache
<VirtualHost *:80>
    ServerName docs.rvbbit.io
    DocumentRoot /path/to/rvbbit/doc_pages

    <Directory /path/to/rvbbit/doc_pages>
        Options Indexes FollowSymLinks
        AllowOverride None
        Require all granted
    </Directory>
</VirtualHost>
```

## Adding New Pages

1. **Copy Template:**
   ```bash
   cp index.html new-page.html
   ```

2. **Update Content:**
   - Change `<title>` tag
   - Update active nav link in sidebar
   - Replace main content section
   - Add to sidebar navigation in all pages

3. **Use Existing Components:**
   - Code blocks: `.code-block` + `.code-block-header` + `.code-block-content`
   - Info boxes: `.info-box` (variants: `.warning`, `.tip`)
   - Tables: Standard HTML `<table>` (auto-styled)
   - Syntax highlighting: `.kw`, `.fn`, `.str`, `.num`, `.cmt`, `.key` spans

4. **Example Code Block:**
   ```html
   <div class="code-block">
     <div class="code-block-header">
       <span class="code-block-lang">YAML</span>
     </div>
     <div class="code-block-content">
       <pre><span class="key">cascade_id</span>: example
   <span class="key">cells</span>:
     - <span class="key">name</span>: demo</pre>
     </div>
   </div>
   ```

5. **Example Info Box:**
   ```html
   <div class="info-box tip">
     <div class="info-box-title">Pro Tip</div>
     <p>Use <code>traits: manifest</code> for dynamic tool selection.</p>
   </div>
   ```

## Maintenance

### Updating Documentation

When codebase changes:

1. **Re-introspect Source Files:**
   - Read updated `cascade.py`, `runner.py`, etc.
   - Verify new fields, options, or behaviors
   - Check examples in `examples/` directory

2. **Update Affected Pages:**
   - Add new configuration options to tables
   - Update code examples with new syntax
   - Add deprecation notices for removed features

3. **Keep Style Consistent:**

   - Follow existing component patterns
