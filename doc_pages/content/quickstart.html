Quickstart Guide

Get RVBBIT running with a working database, sample data, and both the Web UI and SQL server in under 10 minutes. This guide assumes you have Docker and Python 3.10+ installed.

Prerequisites

Before starting, ensure you have:

Requirement Version Check Command
Python 3.10 or higher python --version
Docker 20.10 or higher docker --version
OpenRouter API Key - Get one here
Virtual Environment Recommended

We recommend using a virtual environment to avoid conflicts with other Python packages:

python -m venv venv
source venv/bin/activate  # On Windows: venv\Scripts\activate

Step 1: Install RVBBIT

Bash 
# Install RVBBIT
pip install rvbbit

# Verify installation
rvbbit --version

Optional installation variants:

Bash 
# With local HuggingFace model support
pip install rvbbit[local-models]

# With browser automation (Playwright)
pip install rvbbit[browser]

# Everything
pip install rvbbit[all]

# From source (for development)
git clone https://github.com/ryrobes/rvbbit
cd rvbbit/rvbbit
pip install -e .

Step 2: Start ClickHouse

RVBBIT requires ClickHouse for logging, analytics, and vector search. There is no embedded fallback - this is a required dependency.

Option A: Simple Docker Command (Recommended for Quick Start)

Bash 
# Start ClickHouse with persistent storage
docker run -d --name rvbbit-clickhouse \
  -p 9000:9000 \
  -p 8123:8123 \
  -e CLICKHOUSE_USER=rvbbit \
  -e CLICKHOUSE_PASSWORD=rvbbit \
  -e CLICKHOUSE_DEFAULT_ACCESS_MANAGEMENT=1 \
  -v rvbbit-clickhouse-data:/var/lib/clickhouse \
  clickhouse/clickhouse-server:latest

# Verify it's running
docker ps | grep rvbbit-clickhouse

Option B: Docker Compose (For Full Stack)

For a more complete setup with everything configured:

docker-compose.yml 
version: '3.8'

services:
  clickhouse:
    image: clickhouse/clickhouse-server:24.3-alpine
    container_name: rvbbit-clickhouse
    ports:
      - "8123:8123"   # HTTP interface
      - "9000:9000"   # Native protocol
    environment:
      - CLICKHOUSE_DB=rvbbit
      - CLICKHOUSE_USER=rvbbit
      - CLICKHOUSE_PASSWORD=rvbbit
      - CLICKHOUSE_DEFAULT_ACCESS_MANAGEMENT=1
    volumes:
      - clickhouse-data:/var/lib/clickhouse
    healthcheck:
      test: ["CMD", "clickhouse-client", "--query", "SELECT 1"]
      interval: 10s
      timeout: 5s
      retries: 5

volumes:
  clickhouse-data:
Bash 
# Start with Docker Compose
docker compose up -d

# Check status
docker compose ps
ClickHouse Ports
  • 9000: Native protocol (used by RVBBIT)
  • 8123: HTTP interface (useful for debugging)

Step 3: Initialize Workspace

Create a new RVBBIT workspace with starter files and examples:

Bash 
# Create and enter your project directory
mkdir my-rvbbit-project
cd my-rvbbit-project

# Initialize RVBBIT workspace
rvbbit init .

# Or create in a new directory
rvbbit init my-project

This creates the following structure:

Step 4: Configure Environment

Copy the example environment file and configure your API keys:

Bash 
# Copy the template
cp .env.example .env

# Edit with your favorite editor
nano .env  # or: code .env, vim .env

At minimum, set these values:

.env 
# =============================================================================
# REQUIRED: Workspace Root
# =============================================================================
# IMPORTANT: Set this to your project directory's absolute path!
# All RVBBIT data (logs, states, databases) is stored relative to this path.
RVBBIT_ROOT=/home/user/my-rvbbit-project

# =============================================================================
# REQUIRED: LLM Provider
# =============================================================================
# Get your key at: https://openrouter.ai/keys
OPENROUTER_API_KEY=sk-or-v1-your-key-here

# =============================================================================
# REQUIRED: ClickHouse Database
# =============================================================================
# These match the Docker setup from Step 2
RVBBIT_CLICKHOUSE_HOST=localhost
RVBBIT_CLICKHOUSE_PORT=9000
RVBBIT_CLICKHOUSE_DATABASE=rvbbit
RVBBIT_CLICKHOUSE_USER=rvbbit
RVBBIT_CLICKHOUSE_PASSWORD=rvbbit

# =============================================================================
# OPTIONAL: Model Customization
# =============================================================================
# RVBBIT_DEFAULT_MODEL=anthropic/claude-sonnet-4
# RVBBIT_DEFAULT_EMBED_MODEL=qwen/qwen3-embedding-8b
RVBBIT_ROOT is Important!

RVBBIT_ROOT should point to your project directory's absolute path. All data files (logs, session databases, states) are stored relative to this path. If you run RVBBIT from different directories without setting this, your data may end up scattered.

Automatic Setup

When you run rvbbit init, it automatically creates a .env file with RVBBIT_ROOT set to the workspace's absolute path. Just add your API key!

Keep Your API Key Secret

Never commit your .env file to version control. The .gitignore created by rvbbit init already excludes it.

Step 7: Run Your First Cascade

Test your setup by running the hello_world example:

Bash 
# Run with default input
rvbbit run cascades/examples/hello_world.yaml

# Run with custom input
rvbbit run cascades/examples/hello_world.yaml --input '{"name": "Alice"}'

You should see the cascade execute and produce a greeting.

What Just Happened?

Let's look at the cascade file:

cascades/examples/hello_world.yaml 
cascade_id: hello_world
description: A simple greeting cascade to verify your setup works.

inputs_schema:
  name: The name to greet (optional, defaults to "World")

cells:
  - name: greet
    instructions: |
      Generate a friendly greeting for {{ input.name | default("World") }}.

      Be creative but keep it brief (1-2 sentences).
      Include a fun fact or observation to make it memorable.
    rules:
      max_turns: 1

Key concepts:

  • cascade_id: Unique identifier for this workflow
  • inputs_schema: Documents expected inputs
  • cells: List of execution steps
  • instructions: Jinja2-templated prompt for the LLM
  • {{ input.name }}: Reference to input data

No DuckDB CLI?

Install DuckDB CLI with pip install duckdb or download from duckdb.org. Alternatively, use Python:

Create SQL Connection

Create a connection file for the sample database:

Bash 
# Create sql_connections directory if it doesn't exist
mkdir -p sql_connections

# Create connection file
cat > sql_connections/sample_data.yaml << 'EOF'
# Sample Data Connection
# A DuckDB database with customers, orders, and products for testing

connection_name: sample_data
type: duckdb
database: ./data/sample.duckdb
enabled: true
read_only: true
sample_row_limit: 100
distinct_value_threshold: 50
EOF

Discover Schema

Run schema discovery to index your database:

Bash 
# Crawl and index all connected databases
rvbbit sql crawl

This creates metadata files in sql_connections/samples/ that enable:

  • Schema browsing in the Studio UI
  • SQL autocomplete
  • Semantic search across your schemas

Query Your Data

Test the connection with a query:

Bash 
# Query the sample database
rvbbit sql query "SELECT * FROM sample_data.customers"

# Join tables
rvbbit sql query "
SELECT
  c.name,
  c.company,
  SUM(o.amount) as total_spent
FROM sample_data.customers c
JOIN sample_data.orders o ON c.id = o.customer_id
WHERE o.status = 'completed'
GROUP BY c.name, c.company
ORDER BY total_spent DESC
"

# Use semantic SQL (if you have embedding model configured)
rvbbit sql query "
SELECT * FROM sample_data.products
WHERE description ABOUT 'enterprise features'
"

Step 10: Start SQL Server

RVBBIT includes a PostgreSQL wire-protocol server that lets you connect any SQL client (DBeaver, DataGrip, psql, etc.) and use Semantic SQL operators.

Bash 
# Start the PostgreSQL-compatible SQL server
rvbbit serve sql --port 15432

# Or shorter alias
rvbbit sql server --port 15432

Connect with psql

Bash 
# Connect with psql
psql -h localhost -p 15432 -U rvbbit -d rvbbit

# Run a query
SELECT * FROM sample_data.customers LIMIT 5;

Connect with Any SQL Client

Use these connection settings in DBeaver, DataGrip, or any PostgreSQL client:

Setting Value
Host localhost
Port 15432
Database rvbbit
Username rvbbit
Password rvbbit
Semantic SQL in Any Client

Once connected, you can use RVBBIT's semantic operators directly in your favorite SQL client: 

-- Semantic search
SELECT * FROM sample_data.products
WHERE description ABOUT 'team collaboration';

-- LLM aggregation
SELECT category, summarize(description) AS summary
FROM sample_data.products
GROUP BY category;

Session Persistence

The database name you use when connecting determines whether your session is ephemeral or persistent:

Database Name Behavior Tables Persist?
memory, default, :memory: In-memory, ephemeral No - lost on disconnect
Any other name (e.g., myproject, analytics) File-based, persistent Yes - survives restarts
Persistent Sessions

When you use a custom database name, RVBBIT creates a DuckDB file at $RVBBIT_ROOT/session_dbs/{database_name}.duckdb. Any tables you create will persist across connections and server restarts.

Connection Examples 
# Ephemeral session - tables lost on disconnect
psql -h localhost -p 15432 -U rvbbit -d memory

# Persistent session - tables saved to session_dbs/myproject.duckdb
psql -h localhost -p 15432 -U rvbbit -d myproject

# Another persistent session - tables saved to session_dbs/analytics.duckdb
psql -h localhost -p 15432 -U rvbbit -d analytics

# In persistent sessions, tables survive across connections:
CREATE TABLE metrics (id INT, value FLOAT);
INSERT INTO metrics VALUES (1, 42.5);
# Disconnect and reconnect - table still exists!

Troubleshooting

ClickHouse Connection Errors

Error: Connection refused
# Check if ClickHouse is running
docker ps | grep clickhouse

# Check logs
docker logs rvbbit-clickhouse

# Restart if needed
docker restart rvbbit-clickhouse
Error: Table doesn't exist

Run database initialization:

rvbbit db init

API Key Issues

Error: API key not configured

Ensure your .env file exists and is loaded:

# Check if .env exists
cat .env | grep OPENROUTER

# Or export directly
export OPENROUTER_API_KEY=sk-or-v1-your-key

# Verify
rvbbit doctor

Port Conflicts

Error: Port already in use

If ports 9000 or 5050 are in use:

# Find what's using the port
lsof -i :9000

# Use a different port for ClickHouse
docker run -d --name rvbbit-ch \
  -p 9001:9000 \
  -e RVBBIT_CLICKHOUSE_PORT=9001 \
  clickhouse/clickhouse-server:latest

# Use a different port for Studio
rvbbit serve studio --port 5051
