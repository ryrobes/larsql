MCP Integration

Model Context Protocol (MCP) enables RVBBIT to discover and use tools from external servers. Connect to filesystem servers, databases, APIs, and more using the standardized MCP protocol.

On This Page

What is MCP?

Model Context Protocol is a standardized way for AI applications to discover and use external tools. MCP servers provide tools, resources, and prompts that RVBBIT can automatically integrate.

Type 6 of Six Tool Types

MCP is the sixth tool type in RVBBIT's unified tool system. MCP tools work exactly like Python functions, cascades, or any other tool type - they all register in the same trait registry.

How RVBBIT Uses MCP

  1. Connect to configured MCP servers
  2. Call tools/list endpoint to discover tools
  3. Convert JSON Schema to Python function signatures
  4. Register tools in trait_registry
  5. Tools become available to all cascades

Server Configuration

MCP servers are configured via config/mcp_servers.yaml or the RVBBIT_MCP_SERVERS_YAML environment variable.

Configuration File Structure

config/mcp_servers.yaml
# Filesystem server (stdio)
- name: filesystem
  transport: stdio
  command: npx
  args:
    - -y
    - @modelcontextprotocol/server-filesystem
    - /tmp
  enabled: true

# Brave Search server
- name: brave-search
  transport: stdio
  command: npx
  args:
    - -y
    - @modelcontextprotocol/server-brave-search
  env:
    BRAVE_API_KEY: ${BRAVE_API_KEY}
  enabled: true

# HTTP server example
- name: github
  transport: http
  url: http://localhost:3000/mcp
  headers:
    Authorization: Bearer ${GITHUB_TOKEN}
  enabled: false

Configuration Fields

FieldRequiredDescription
name✓Server identifier
transport✓stdio or http
enabledEnable/disable server (default: true)
timeoutConnection timeout in seconds (default: 30)

Two Transport Types

1. stdio Transport (Subprocess)

Spawns a local process and communicates via stdin/stdout using JSON-RPC. Best for local tools and npm packages.

stdio Configuration
- name: filesystem
  transport: stdio
  command: npx  # Command to run
  args:  # Command arguments
    - -y
    - @modelcontextprotocol/server-filesystem
    - /tmp  # Allowed directory
  env:  # Environment variables
    DEBUG: "1"
  enabled: true

stdio Fields

FieldDescription
commandExecutable to run (e.g., npx, python, node)
argsArray of command arguments
envEnvironment variables (supports ${VAR} expansion)

2. HTTP Transport (Remote Server)

Connects to a remote MCP server via HTTP POST requests. Best for shared services and remote APIs.

HTTP Configuration
- name: remote-service
  transport: http
  url: https://api.example.com/mcp  # MCP endpoint
  headers:  # Optional headers
    Authorization: Bearer ${API_TOKEN}
    X-Custom-Header: value
  timeout: 60
  enabled: true

HTTP Fields

FieldDescription
urlMCP server endpoint URL
headersHTTP headers (supports ${VAR} expansion)

Tool Discovery Process

When RVBBIT starts or refreshes the tool manifest, it discovers MCP tools automatically:

  1. Load Configuration
    • Read config/mcp_servers.yaml
    • Filter for enabled servers
  2. Connect to Servers
    • stdio: Spawn subprocess
    • HTTP: Establish connection
  3. Introspect Capabilities
    • Call tools/list endpoint
    • Call resources/list endpoint
    • Call prompts/list endpoint
  4. Schema Conversion
    • Convert JSON Schema → Python type hints
    • Build inspect.Signature objects
  5. Wrapper Generation
    • Create callable wrapper function
    • Wrapper calls MCPClient.call_tool()
  6. Registration
    • Register in trait_registry with [MCP] prefix
    • Tools now available to all cascades

Schema Conversion

MCP tools use JSON Schema. RVBBIT converts this to Python signatures:

JSON Schema → Python
# MCP Tool Schema (JSON):
{
  "name": "read_file",
  "description": "Read contents of a file",
  "inputSchema": {
    "type": "object",
    "properties": {
      "path": {"type": "string", "description": "File path"}
    },
    "required": ["path"]
  }
}

# Converted to Python Signature:

    # Wrapper calls MCPClient.call_tool("read_file", {"path": path})

CLI Commands

List Configured Servers

List Servers
rvbbit mcp list

# Output:
# Name              Transport  Status   Tools
# ──────────────────────────────────────────
# filesystem        stdio      enabled  5
# brave-search      stdio      enabled  3
# github            http       disabled 0

Check Server Status

Status Check
rvbbit mcp status

# Shows health and connection status for all servers

Introspect Server Tools

Introspect
rvbbit mcp introspect filesystem

# Lists all tools from the filesystem server:
# - read_file: Read contents of a file
# - write_file: Write contents to a file
# - list_directory: List files in directory
# - search_files: Search for files
# - get_file_info: Get file metadata

Test Tool Execution

Test Tool
rvbbit mcp test filesystem read_file --args '{"path": "/tmp/test.txt"}'

# Executes the tool and shows the result

Add New Server

Add Server
# Add stdio server
rvbbit mcp add my-server stdio \
  --command "npx -y @user/mcp-server" \
  --env "API_KEY=\${MY_API_KEY}"

# Add HTTP server
rvbbit mcp add remote-api http \
  --url "https://api.example.com/mcp" \
  --env "Authorization=Bearer \${TOKEN}"

Refresh Tool Discovery

Refresh
rvbbit mcp refresh

# Re-discovers tools from all enabled servers
# Updates trait manifest

Using MCP Tools in Cascades

Explicit Tool List

Using MCP Tools
- name: read_config
  instructions: "Read the config file and summarize it"
  traits:
    - read_file  # MCP tool from filesystem server
    - write_file

Manifest (Auto-Discovery)

Manifest Mode
- name: file_operations
  instructions: |
    Process files in /tmp directory.
    Use filesystem tools as needed.
  
  traits: manifest  # Quartermaster sees all MCP tools!

Deterministic Cell with MCP Tool

Deterministic MCP Usage
- name: load_file
  tool: read_file  # MCP tool, direct invocation
  tool_inputs:
    path: "{{ input.file_path }}"
  on_error: auto_fix

Example Integrations

Example 1: Filesystem Operations

Filesystem Server
# config/mcp_servers.yaml
- name: filesystem
  transport: stdio
  command: npx
  args: ["-y", "@modelcontextprotocol/server-filesystem", "/workspace"]
  enabled: true

# In cascade:
- name: process_files
  instructions: |
    List all Python files in /workspace.
    Read each file and identify TODOs.
  
  traits:
    - list_directory  # MCP tool
    - read_file       # MCP tool

Progress Logging

All MCP operations are logged to unified_logs for observability:

  • Server connection attempts
  • Tool discovery operations
  • Tool execution (with duration and status)
  • Errors and timeouts
Query MCP Logs
-- View MCP activity
SELECT
  timestamp,
  role,
  content,
  metadata
FROM all_data
WHERE role = 'mcp_progress'
ORDER BY timestamp DESC
LIMIT 20;

Best Practices

Security

  • Limit Access: For filesystem servers, specify allowed directories
  • Use Environment Variables: Never hardcode API keys in config
  • Disable Unused Servers: Set enabled: false for servers you're not using
  • Validate Inputs: Use wards to validate MCP tool inputs/outputs

Performance

  • Connection Pooling: stdio servers spawn once and stay alive
  • Timeouts: Set appropriate timeouts for slow operations
  • Selective Discovery: Only enable servers you need

Debugging

  • Test First: Use rvbbit mcp test before using in cascades
  • Check Logs: Query all_data WHERE role='mcp_progress'
  • Introspect: Use rvbbit mcp introspect to see available tools
Server Availability

stdio servers must be installed (e.g., via npm). Use npx -y for automatic installation, or install globally first with npm install -g @modelcontextprotocol/server-filesystem.
