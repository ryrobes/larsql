# A Cascade is a sequence of cells with receipts
cascade_id: fraud_assessment_with_candidates
inputs_schema:
  customer_id: "Customer ID"
  transaction_amount: "Transaction amount"
cells:
  - name: assess_risk
    instructions: |
      Return ONLY valid JSON with:
      { risk_score, risk_level, reasoning, recommended_action, confidence }
    candidates:
      factor: 3
      mode: evaluate
      evaluator_instructions: |
        Pick the most conservative valid assessment.
    wards:
      post:
        - validator:
            python: |
              # validate JSON contract; retry if invalid
          mode: retry
          max_retries: 2
    handoffs: [finalize]
# Map a Cascade over an array of inputs (data-driven parallelism)
inputs = [{"customer_id": 1, "amount": 120}, {"customer_id": 2, "amount": 99999}]

results = rvbbit.map_cascade(
  cascade="cascades/fraud_assessment_with_candidates.yaml",
  inputs=inputs,
  concurrency=8,
  cache="aggressive",
  receipts=True,
)

# Each result includes outputs + receipts + deep links
print(results[0].output, results[0].receipt)

Core Concepts

RVBBIT is a visual IDE for building LLM cascades. Chain polyglot cells, run parallel candidates, and get detailed execution receipts‚Äîall in a flat, query-able interface.

Cascades

A sequence of cells with explicit inputs/outputs and deterministic control flow. Chain LLM cells, code execution, and trait invocations with handoffs between cells.

Flow cells ‚Ä¢ handoffs ‚Ä¢ context

Cells

The building blocks. Each cell can be an LLM call, SQL query, Python script, or direct trait invocation. Mix and match freely within a cascade.

Polyglot LLM ‚Ä¢ SQL ‚Ä¢ Python ‚Ä¢ JS

Candidates

Run a cell multiple times in parallel (clones or variants), then evaluate or aggregate results. Great for reliability without retry loops.

Parallel factor ‚Ä¢ evaluate ‚Ä¢ aggregate

Traits

Reusable tools that cells can invoke‚Äîfrom shell commands to custom Python functions to nested cascades. Build a library of traits for your domain.

Reuse tools ‚Ä¢ functions ‚Ä¢ cascades

Receipts

Detailed execution records at every level‚Äîcell, candidate, run, cascade. Cost, latency, model mix, cache hits. Query-able from SQL.

Observability cost ‚Ä¢ time ‚Ä¢ lineage

SQL Interop

Call cascades as SQL UDFs, map over table rows, and query execution receipts. LLM workflows become first-class database citizens.

Integration UDF ‚Ä¢ mapping ‚Ä¢ queries

Visual Studio

Build cascades in a clean, linear IDE. Author cells, run inline, inspect outputs, and debug receipts‚Äî without the spaghetti of traditional flow editors.

Key Features

Inline Execution

Run cells individually or all at once. See outputs, receipts, and errors inline without switching views.

Candidate Tabs

When a cell runs multiple candidates, see all results as tabs with individual receipts.

Receipt Browser

Drill into cost, latency, model usage, and cache hits at every level of execution.

SQL Interop

Call cascades from SQL as UDFs, map over table rows, and query execution receipts. LLM workflows become database-native with caching and cost tracking built in.

Cascades as Functions

Use rvbbit_udf() for simple LLM calls or rvbbit_cascade_udf() for full cascades. Map over rows, aggregate results, and get detailed receipts for every invocation.

SQL
Query receipts: Every cascade execution generates queryable receipts. Track costs, cache hits, and model usage directly from SQL.
