Mental model
Cascade = program Cell = step Traits = tools Outputs = saved results

Cell contracts

Typed shapes and validators keep outputs deterministic and repairable.

Receipts drill-down

From receipts to run detail to candidate context and validator decisions.

YAML Cell with Candidates + Wards 
cells:
  - name: classify
    traits: [llm, sql]
    candidates:
      count: 5
      merge: best
    wards:
      validate: schema:product_tag
      on_fail: repair_then_retry
    outputs:
      - labels
SQL UDF calling a Cascade 
SELECT
  rvbbit_cascade_udf(
    'product_enrichment',
    to_jsonb(row)
  ) AS enriched
