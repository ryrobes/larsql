This is what you write

A SQL query. That's it.

query.sql 
SELECT
    author,
    text EXTRACTS 'urls' AS urls,
    author_name ASK 'is this a notable AI influencer? score 0-1' AS notability,
    summarize_urls(text) AS url_summaries,
    TLDR(text) AS summary
FROM tweets
WHERE text IMPLIES 'bullish on AI development'
  AND text CONTRADICTS 'AI will replace all developers'

This is how you extend it

Create custom SQL operators with YAML. No code required. Drop a file, restart, your operator works.

cascades/semantic_sql/sounds_like.yaml 
cascade_id: semantic_sounds_like

sql_function:
  name: sounds_like
  operators:
    - "{{ text }} SOUNDS_LIKE {{ reference }}"
  returns: BOOLEAN

cells:
  - name: evaluate
    model: gemini-2.5-flash-lite
    instructions: |
      Do these sound phonetically similar?
      Text: {{ input.text }}
      Reference: {{ input.reference }}
      Return only true or false.
