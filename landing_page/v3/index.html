Returns the 20 most similar products. Instant.

The hybrid trick: 10,000x cost reduction

Vector search is fast but imprecise. LLM judgment is precise but expensive. Combine them. 

WITH candidates AS (
  SELECT * FROM VECTOR_SEARCH('affordable eco products', 'products', 100)
)
SELECT * FROM candidates
WHERE description MEANS 'genuinely eco-friendly, not greenwashing'
LIMIT 10;
🔍 Vector search: 1M rows → 100 candidates (fast, cheap)
🧠 LLM evaluation: 100 → 10 winners (smart, still cheap)

Ask questions, not just queries.

You ran a query. Now you want to know why. ANALYZE hands your results to an agent with SQL access, Python access, and your full context—then writes its findings to a table you can query later.

1 Query runs in the background
2 Results + question go to an agent cascade
3 Agent investigates (can run more SQL, Python, etc.)
4 Analysis saved to analyze table
SQL Fire and forget 
ANALYZE 'Why did churn spike in Q3?'
SELECT month, churn_rate, segment
FROM metrics
WHERE year = 2024;
Later, check the analysis
SELECT * FROM analyze WHERE question LIKE '%churn%';
Bring your own agent
ANALYZE 'Why did churn spike in Q3?' WITH CALLIOPE
SELECT month, churn_rate, segment FROM metrics;

Any cascade can be the analysis engine. Domain expert? Compliance bot? Your call.

The magic trick behind the operators.

You've been using cascades this whole time. That MEANS operator? It's a YAML file.

YAML cascades/semantic_sql/matches.cascade.yaml 
cascade_id: semantic_matches

sql_function:
  operators:
    - "{{ text }} MEANS {{ criterion }}"

cells:
  - name: evaluate
    model: gemini-2.5-flash-lite
    instructions: |
      Does this text match the criterion?
      Text: {{ input.text }}
      Criterion: {{ input.criterion }}

Every operator is a file.

Want to change the model? Edit the file. Want stricter validation? Add a ward. Want three models to compete? Add candidates.

The same system powers everything.

A simple MEANS check (1 cell) and a fraud detection workflow (7 cells, 3 models) are both cascades. Same primitives, different depths.

No abstraction gap.

Analysts use operators. Engineers customize cascades. Same system, different entry points.

Need a custom operator? It's just YAML.

Drop a file. Restart. Your operator works. No code changes required.

YAML cascades/sounds_like.yaml 
cascade_id: semantic_sounds_like

sql_function:
  name: sounds_like
  operators:
    - "{{ text }} SOUNDS_LIKE {{ ref }}"
  returns: BOOLEAN

cells:
  - name: evaluate
    model: gemini-2.5-flash-lite
    instructions: |
      Do these sound phonetically similar?
      Text: {{ input.text }}
      Reference: {{ input.ref }}
      Return only true or false.
